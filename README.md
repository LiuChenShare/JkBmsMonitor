# 极空(JK)保护板 BLE 监控 —— 工程目录

逐步把"通过蓝牙读极空保护板数据、用 Web 面板监控"做出来。
技术栈:C#(.NET 10) 全栈。

## 当前状态:第 1+2 步 —— 数据通路 + JK02 协议解析

`JkBmsMonitor/` 是一个控制台程序,做四件事:

1. 扫描附近 BLE 设备,标记名称含 `JK` 的(你的极空板)
2. 连接后打印完整 GATT 服务/特征值清单(**拿到真实 UUID**)
3. 订阅所有 Notify 特征值,实时收数据
4. 把分包通知**重组为 300 字节完整帧** → 解析 **JK02 电芯信息帧** →
   直接打印结构化电池数据(总电压/电流/SOC/温度/各串电压内阻/压差 ...)

解析逻辑独立在 `JkBmsProtocol.cs`,帧头校验 `55 AA EB 90`、CRC 用累加和,
字段偏移取自权威实现 [syssi/esphome-jk-bms](https://github.com/syssi/esphome-jk-bms)
的 `decode_jk02_cell_info_()`。非电芯帧(设置帧 0x01 / 设备信息帧 0x03 等)
目前回退为头部 hex,留作后续扩展。

## 运行

```bash
cd D:/AI/闹着玩/jikong/JkBmsMonitor
dotnet run -c Release
```

> 必须在**带蓝牙的 Windows 10/11** 上跑。运行时蓝牙要保持开启。

操作:
- 程序自动扫描,列表里找到你的极空板(带 `<== JK` 标记),输序号回车
- 连上后会打印一坨 `Service / Characteristic`
- 之后收到通知就重组+解析,解析成功的电芯信息帧会打印成这样的电池数据块:
  ```
  [14:32:07.218] ✅ 解析成功:
  ┌─────────────── 电池实时数据 ───────────────
  │ 总电压 :   53.240 V        电流 :  -12.300 A
  │ 功率   :  -655.35 W  (放电)
  │ SOC    :       78 %        SOH  :      100 %
  ...
  ```

## 看到解析结果后,下一步

- 如果数值对得上(电压/电流/串数合理),协议就稳了
- 个别字段偏移对不上的话:把打印的电池数据块 + 一行完整 300 字节帧 hex 贴给我,
  我对照 syssi 源码调偏移
- 之后进第 3 步:Worker Service + 自动重连 + 主动请求推送(部分板子不发,需写命令触发)
- 第 4 步:SignalR + SQLite + Blazor 做面板

## 已知坑

- **未配对**:某些板子需先在「Windows 蓝牙设置」里和保护板配对,否则连接失败/拿不到服务
- **拿到服务失败(NotReachable)**:重启电脑蓝牙,或把板子断电重上
- **设备名 "(未知名称)"**:可接受,靠 MAC 地址选;或先在系统里配对一次
- **断流**:程序尽量保活了 GATT 对象,但 BLE 偶尔会掉,断开重连即可(后续会加重连逻辑)
